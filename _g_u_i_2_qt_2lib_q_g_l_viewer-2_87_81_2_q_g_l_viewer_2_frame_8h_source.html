342  // A frame is as a new coordinate system, defined with respect to a reference
343  // frame (the world coordinate system by default, see the "Composition of
344  // frame" section).
345 
346  // The transformOf() (resp. inverseTransformOf()) functions transform a 3D
347  // vector from (resp. to) the world coordinates system. This section defines
348  // the 3D vector transformation functions. See the Coordinate system
349  // transformation of 3D points above for the transformation of 3D points. The
350  // difference between the two sets of functions is simple: for vectors, only
351  // the rotational part of the transformations is taken into account, while
352  // translation is also considered for 3D points.
353 
354  // The length of the resulting transformed vector is identical to the one of
355  // the source vector for all the described functions.
356 
357  // When local is prepended to the names of the functions, the functions simply
358  // transform from (and to) the reference frame.
359 
360  // When In (resp. From) is appended to the names, the functions transform from
361  // (resp. To) the frame that is given as an argument. The frame does not need
362  // to be in the same branch or the hierarchical tree, and can be \c NULL (the
363  // world coordinates system).
364 
365  // Combining any of these functions with its inverse (in any order) leads to
366  // the identity.

372  Vec transformOfIn(const Vec &src, const Frame *const in) const;
373  Vec transformOfFrom(const Vec &src, const Frame *const from) const;
