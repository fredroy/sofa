31 /* DefaultMultiMatrixAccessor is a simplest class managing the global matrix setup.
32  * This class allow only cases where there are several Mechanical State on simulation scene
33  * and interactions between them.
34  *
35  * CRSMultiMatrixAccessor is a more powerfull class managing the global system matrix.
36  * This class allow the scene where there are mappings so mapped Mechanical State. It compute
37  * The contribution of stiffness on mapped Mechanical State to the root State related by mapping.
38  *
39  * see Sofa/doc/modules/linearsolver/LinearSolver.tex
40  * see all scenes in example/Component/linearsolver/MatrixContribution* for tests
41  *
42  * */
