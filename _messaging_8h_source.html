22 /***********************************************************************
23  * Not so Simple Messaging API
24  *
25  *
26  * Eg:
27  * If you add in your source code:
28  * msg_info(this) << "This is an error messaging from somewhere in my sofa"
29  * " component saying that he cannot read a file with "
30  * " name [" << filename << "]".
31  *
32  * This should print (using the Sofa pretty formatter) something like:
33  * [INFO] (name(component)): This is an error messae from somewhere in my sofa
34  * component saying that he cannot read a file with
35  * a name [/home/path/to/my/file.txt]
36  * source: /home/path/to/mysourceode.cpp:354:0
37  *
38  * Eg:
39  * msg_info() << "This is the same as the version with 'this' ;
40  *
41  * Eg:
42  * msg_info("EmitterBasedOnString") << "An emitter's name" ;
43  *
44  *
45  * The API is composed of two objects:
46  * - MessageHandler
47  * - Message
48  *
49  * a messaging contains:
50  * - a textual description (not formatted).
51  * - some specifier about the messaging type
52  * - a (possibly empty) backtrace
53  * - the source file and line from where the messaging was emitted.
54  *
55  * for any usefull purpose, a messaging needs to be send to
56  * a MessageHandler either using an object or using the singleton/static
57  * part of the API.
58  *
59  * The default messaging is adding the messaging to a list and gives
60  * them a number. This number can then be used to retrieve the messaging
61  * in the messaging list. Once added, the messaging also call the
62  * virtual prettyPrintMessage() for effective displaying the messaging.
63  *
64  * It is possible, by inheriting and overriding the prettyPrintMessage
65  * function to implement different policy.
66  * Currently we are implementing
67  * SofaStyleHandler to format the messaging in a sofa way
68  * (eg: [WARN] (object<classType>): messaging)
69  * and print it on std::cout
70  * ClangStyleHandler: format the messaging in a clang style
71  * to easy integration of the messaging in the IDE
72  * (clicking on the messaging to go to the source code).
73  *
74  * This is the place to go to implement other policy like:
75  * open a QTDialogBox, log the messaging into a file, report the
76  * messaging into the serr/sout of sofa.
77  **********************************************************************/
